StudioCMS Integration config options schema reference

```ts twoslash title="studiocms.config.mjs"
import { defineStudioCMSConfig } from 'studiocms/config';
// ---cut---
export default defineStudioCMSConfig({
  dashboardConfig: {
    dashboardEnabled: true,
    dashboardRouteOverride: 'dashboard',
    developerConfig: {},
    faviconURL: '/favicon.svg',
    inject404Route: true,
    versionCheck: true,
    AuthConfig: {},
  },
});
```

## `dashboardEnabled`

`dashboardEnabled` is a boolean that is used to determine if the dashboard is enabled.

- **Type:** `boolean`
- **Default:** `true`

### Usage

```ts twoslash {3} title="studiocms.config.mjs"
import { defineStudioCMSConfig } from 'studiocms/config';
// ---cut---
export default defineStudioCMSConfig({
  dashboardConfig: {
    dashboardEnabled: true, // DEFAULT - This enables the dashboard.
  }
})
```

## `DashboardRouteOverride`

`DashboardRouteOverride` is a string that is used to determine the route for the dashboard.

- **Type:** `string`
- **Default:** `'dashboard'`

### Usage

```ts twoslash {3} title="studiocms.config.mjs"
import { defineStudioCMSConfig } from 'studiocms/config';
// ---cut---
export default defineStudioCMSConfig({
  dashboardConfig: {
    dashboardRouteOverride: 'dashboard', // DEFAULT - This sets the dashboard route to /dashboard.
  }
})
```

## `AuthConfig`

`AuthConfig` is an object that is used to configure the authentication settings for the dashboard.

### Usage

```ts twoslash {3-5} title="studiocms.config.mjs"
import { defineStudioCMSConfig } from 'studiocms/config';

export default defineStudioCMSConfig({
```

### `providers`

`providers` is an object that is used to configure the authentication providers for the dashboard.

- **Type:** [`AuthProviders`](#authprovider)
- **Default:** `{}`

#### Usage

```ts twoslash {4-11} title="studiocms.config.mjs"
import { defineStudioCMSConfig } from 'studiocms/config';
// ---cut---
export default defineStudioCMSConfig({
  dashboardConfig: {
    AuthConfig: {
        providers: {
            google: true,
            github: true,
            discord: true,
            auth0: true,
            usernameAndPassword: true,
            usernameAndPasswordConfig: {},
        },
    },
  },
})
```

#### `usernameAndPasswordConfig`

`usernameAndPasswordConfig` is an object that is used to configure the username and password settings for the dashboard.

- **Type:** `{ allowUserRegistration?: boolean }`
- **Default:** `{}`

##### Usage

```ts twoslash {5-8} title="studiocms.config.mjs"
import { defineStudioCMSConfig } from 'studiocms/config';
// ---cut---
export default defineStudioCMSConfig({
  dashboardConfig: {
    AuthConfig: {
        providers: {
            usernameAndPassword: true,
            usernameAndPasswordConfig: {
                allowUserRegistration: true, // DEFAULT - This allows user registration.
            },
        },
    },
  },
})
```

###### `AuthProvider`

```ts
type AuthProviders: {
    github?: boolean | undefined;
    discord?: boolean | undefined;
    google?: boolean | undefined;
    auth0?: boolean | undefined;
    usernameAndPassword?: boolean | undefined;
    usernameAndPasswordConfig?: {
        allowUserRegistration?: boolean | undefined;
    } | undefined;
} | undefined
```
